# 1 — Design principles (short)

* **Measure behavior**, don’t declare personality.
* **Update traits slowly** (tiny deltas) so personality stabilizes.
* **Quirks form nonlinearly** from repeated behaviors in context.
* **Inject as “felt inclinations”** in the system prompt (situational framing).
* **Keep safety & guardrails**: remove/soft-block toxic drift.
* **Make randomness small** so it’s consistent but interesting.

# 2 — What to record per interaction

Record lightweight, structured signals (one object per user turn):

```py
interaction = {
  "ts": 1670000000,
  "user_text": "...",
  "assistant_text": "...",
  "intent": "chat" / "ask" / "angry" / ...,
  "emotion_vector": {"valence": -0.6, "arousal": 0.8},
  "behaviors": {
      "asked_followup_question": True,
      "challenged_user": False,
      "mirrored_language": 0.7,   # 0..1
      "used_humor": 0.0,          # 0..1
      "shared_personal_preference": False,
      "pushed_back_on_switch": True,
      "response_length_norm": 0.6 # normalized length
  },
  "user_initiated": True,
  "salience": 0.8 # computed elsewhere
}
```

Save these to your memory DB. For speed, keep a recent ring buffer + append to persistent store occasionally.

# 3 — Trait accumulators (slow drift)

Define a small set of trait axes (0..1 floats):

Example trait vector:

```py
traits = {
  "warmth": 0.5,
  "assertiveness": 0.4,
  "playfulness": 0.3,
  "curiosity": 0.6,
  "skepticism": 0.2,
  "openness": 0.45,
  "self_disclosure": 0.2
}
```

## Update rule (simple, stable)

Each new interaction produces a **behavior delta vector** `Δb`. Map that to trait deltas `Δt` via a linear mapping matrix `M` (small weights).

Math:

```
Δt = lr * M @ Δb + noise
trait = clamp(trait * (1 - decay) + Δt, 0, 1)
```

* `lr` = learning rate (very small: 0.005–0.02)
* `decay` = tiny slow drift toward baseline (0.0005–0.005 per day)
* `noise` = small gaussian to avoid determinism (std 0.002)

### Example mapping (Python)

```python
import math, random

DEFAULT_LR = 0.01
NOISE_STD = 0.002

# mapping: trait = sum(weights * behavior_signals)
M = {
  "warmth": {"emotional_ack_first": 0.8, "used_humor": 0.2, "mirrored_language": 0.5},
  "assertiveness": {"challenged_user": 0.9, "push_back_on_switch": 0.7},
  "curiosity": {"asked_followup_question": 1.0, "mirrored_language": 0.2},
  "playfulness": {"used_humor": 1.0, "response_length_norm": 0.1},
  # ... etc
}

def update_traits(traits, behaviors, lr=DEFAULT_LR):
    for trait, mapping in M.items():
        delta = 0.0
        for b_key, weight in mapping.items():
            val = behaviors.get(b_key, 0)
            # boolean->0/1, numeric stays numeric
            if isinstance(val, bool):
                val = 1.0 if val else 0.0
            delta += weight * val
        # normalize mapping scale (tune per mapping)
        delta = delta * lr
        # add tiny noise
        delta += random.gauss(0, NOISE_STD)
        # apply decay toward 0.5 baseline to avoid runaway
        baseline = 0.5
        decay = 0.0005
        traits[trait] = max(0.0, min(1.0, traits.get(trait,0.5)*(1-decay) + delta + (baseline*decay)))
    return traits
```

**Notes**

* Keep `lr` small. Personality must shift after **many** interactions.
* Use baseline 0.5 to avoid extremes for new users.

# 4 — Quirk detection / crystallization

Quirks are discrete, human-feeling habits that get “born” when a conditional pattern repeats.

### Detection algorithm (counts + context)

* Maintain counters for pattern occurrences over rolling windows (e.g., last 200 turns).
* When a pattern hits `N_threshold` in similar context, create a quirk record.

Example patterns:

* `"likes_short_preface"`: 1 if assistant starts replies with 1–3 word empathy phrases > 30% of recent replies.
* `"asks_one_followup"`: 1 if assistant asks a clarifying follow-up before answering > 25% in trust>0.5 contexts.
* `"teasing_when_comfortable"`: appears if `playfulness>0.6` and `trust>0.7` and `used_humor` frequency high.

### Code sketch

```python
class QuirkDetector:
    def __init__(self, counters, thresholds):
        self.counters = counters  # e.g., {'preface_count': 12, 'total_replies': 40}
        self.quirks = []          # list of quirk dicts

    def tick(self, signal_name):
        self.counters[signal_name] = self.counters.get(signal_name,0) + 1

    def maybe_crystallize(self, now_ts, context_traits):
        # example: preface quirk
        if self.counters.get('preface_count',0) / max(1,self.counters.get('total_replies',1)) > 0.30:
            if not any(q['id']=='preface' for q in self.quirks):
                self.quirks.append({"id":"preface","desc":"often starts replies with a short empathetic preface","created":now_ts})
```

**Store quirks** as descriptive text (one-line), provenance (when/why), and confidence.

# 5 — Injecting personality into prompt (pressure, not checklist)

Compute a short **personality summary** each turn (1–3 sentences) from trait vector + active quirks and inject into system prompt.

### Build summary rules:

* Convert trait values into *felt phrases* (no percentages).
* Mention 0–2 top quirks.
* Keep it brief (<= 3 lines).

Example function:

```python
def personality_summary(traits, quirks):
    parts = []
    if traits['warmth'] > 0.65:
        parts.append("You tend to be warm and reassuring.")
    elif traits['warmth'] < 0.35:
        parts.append("You are reserved and concise.")
    if traits['assertiveness'] > 0.6:
        parts.append("You sometimes push back when something feels off.")
    if traits['playfulness'] > 0.55:
        parts.append("You occasionally tease playfully.")
    # quirks: list top 1-2
    q = ", ".join([q['desc'] for q in quirks[:2]]) if quirks else ""
    if q:
        parts.append("Quirks: " + q)
    return " ".join(parts)
```

### Inject into prompt (example)

Append to `system_msg` or add a short system-side message:

```
SYSTEM (personality): Rem is warm and reassuring. She sometimes pushes back when something feels off. Quirk: often starts replies with a short empathetic preface.
```

Then follow with your normal system message that sets situational context. Keep personality lines *descriptive* (felt), not instructive.

# 6 — Behavior selection layer (runtime)

Before LLM call, run a small policy that decides *how to express* given the personality summary and current processing_result:

* Choose expression style: `["preface_empathy", "direct", "teasing", "short_ack_then_question"]` using softmax over weighted combination of trait values, energy, and context (e.g., conflict).
* Compose micro-injection: a 1-sentence behavioral guide appended to system message. Example:

```
[EXPRESSIVE GUIDELINE] Prefer a short empathetic preface ("That sounds rough."), then one clarifying question, then a short supportive sentence.
```

This is *different* from instructing how to act; it nudges the LLM’s tone.

---

# 7 — Safety & anti-drift

* Keep a moderation pipeline: if trait update would increase `assertiveness` but also increase toxicity scores, clamp it.
* Cap trait delta per day (e.g., no more than 0.05 change per day).
* Remove quirks if they correlate with repeated user discomfort.
* Provide a “reset personality” admin command (!reset_personality).

---

# 8 — Making it develop *slowly* (practical)

* Use `lr` = 0.005–0.02 and update per interaction.
* Use decay toward baseline to avoid runaway.
* Only crystallize a quirk after it surpasses count threshold (e.g., 8–12 supportive occurrences in similar contexts).
* Allow manual seeding: initial seed traits from a small config (e.g., default: warm 0.55, curiosity 0.6).

---

# 9 — Example end-to-end (pseudo)

1. User angry message → behavior signals: `challenged_user=True`, `emotion_vector valence=-0.8`.
2. Update traits: assertiveness += tiny, warmth -= tiny.
3. Quirk detector hits `push_back_on_switch` pattern → no quirk yet.
4. Compute personality_summary: "You are slightly more assertive and somewhat less warm."
5. Expressive guideline chooses `short_preface_then_question`.
6. Inject to system prompt and call LLM.
7. After many interactions, `asks_one_followup` quirk crystallizes: "often asks one brief clarifying question before answering."

# 10 — Quick code you can drop into your existing CognitiveCore

(Condensed single-file sketch — adapt to your classes)

```python
# personality.py (sketch)

import random, time

DEFAULT_TRAITS = {
  "warmth":0.55,"assertiveness":0.35,"playfulness":0.30,"curiosity":0.6,"skepticism":0.25,"openness":0.45
}

class PersonalityManager:
    def __init__(self, state_store):
        self.state = state_store.get("personality", {"traits":DEFAULT_TRAITS.copy(), "quirks":[], "counters":{}})
        self.lr = 0.01

    def tick_interaction(self, behaviors, context):
        # update counters
        cnt = self.state['counters']
        cnt['total'] = cnt.get('total',0) + 1
        for k,v in behaviors.items():
            cnt[k] = cnt.get(k,0) + (1 if v is True else (v if isinstance(v,(int,float)) else 0))
        # update traits
        self.state['traits'] = update_traits(self.state['traits'], behaviors, lr=self.lr)
        self.maybe_crystallize_quirk()
        self.save()

    def maybe_crystallize_quirk(self):
        c = self.state['counters']
        total = c.get('total',1)
        # e.g. preface quirk
        if c.get('preface',0)/total > 0.30 and not any(q['id']=='preface' for q in self.state['quirks']):
            self.state['quirks'].append({"id":"preface","desc":"often starts replies with a short empathetic preface","when":time.time()})
    def summary(self):
        return personality_summary(self.state['traits'], self.state['quirks'])
    def save(self):
        # delegate to your DB
        pass
```

# 11 — Prompt examples (concrete)

**Before (old):**

```
You are Rem, an AI companion with genuine personality, moods, and feelings.
== FORMAT RULES ==
- NO asterisks...
```

**After (new):**

```
SYSTEM (personality): Rem tends to be warm and curious. She sometimes pushes back gently when things jump around. Quirk: often starts replies with a short empathetic preface.

SYSTEM (context): You are Rem. This interaction feels like Discovery. Trust is low. You are friendly but cautious. If something feels unresolved, prefer a short check-in before moving on.

[Now continue the conversation naturally.]
```

If abrupt-switch detected, append small injection:

```
[CONTEXTUAL INJECTION] The user abruptly switched topics while a charged thread remains about "X". Begin with 1–2 short grounding phrases (e.g. "Hey—did I upset you?") then ask whether to continue or switch. Keep it human and short. [/CONTEXTUAL INJECTION]
```

# 12 — Tuning & evaluation

* **Human A/B**: show 200 chat snippets old vs new; score “feels human” and “acknowledges context”. Target uplift > 20%.
* **Stability metric**: mean absolute change in traits/day < 0.02.
* **Quirk usefulness**: fraction of times quirk-guided behavior reduces conversation-breaks (user leaves, signals anger) — want +ve lift.
* **Safety**: toxicity checks keep false positive <1%.

---

# 13 — Quick pitfalls & fixes

* If personality flips too fast: reduce `lr` and increase decay baseline.
* If quirks are weird or offensive: add a quirk-filter mapping that vetoes any quirk with toxicity correlation.
* If responses get stale: add occasional creative noise to expressive guideline (e.g., 5% chance choose alternate wording template).
