# 1) Smarter memory retrieval — score, filter, diversify

Goal: retrieve only the **most relevant** and **relationship-relevant** memories, not raw recent lines.

Pipeline (high level):

1. Create a retrieval query from the current message + inferred intent + emotional state.
2. Vector search in your DB with metadata filters (phase, tags, recency).
3. Re-rank top-N with a salience function combining: semantic similarity, recency, emotional valence, frequency, and mutuality (how often this was brought up by user vs by bot).
4. Apply MMR (maximal marginal relevance) or diversity penalty to avoid redundant chunks.
5. Return a compact set of memory snippets with provenance + confidence.

Salience scoring (idea):

```
salience = w_sim * sim_score
         + w_recency * recency_boost
         + w_valence * abs(emotional_valence)  # high emotional valence -> more salient
         + w_freq * log(1 + mention_count)
         + w_mutual * mutuality_score   # higher if memory was initiated by user
```

Concrete pseudo-code (Python-ish):

```python
def compute_salience(mem, query_embedding, now_ts, weights):
    sim = cosine(mem.embedding, query_embedding)       # 0..1
    age_hours = (now_ts - mem.timestamp) / 3600.0
    recency = math.exp(-age_hours / weights['recency_half_life'])
    valence = abs(mem.emotional_valence or 0.0)        # 0..1
    freq = math.log1p(mem.mention_count or 0)
    mutual = mem.initiated_by_user and 1.0 or 0.2

    return (weights['w_sim']*sim +
            weights['w_recency']*recency +
            weights['w_valence']*valence +
            weights['w_freq']*freq +
            weights['w_mutual']*mutual)
```

Practical tips:

* Use a small `N=10` from vector DB then re-rank to pick top 3–5.
* Store metadata: `phase`, `type` (identity/episodic/stm), `valence`, `initiated_by_user`, `mention_count`, `timestamp`.
* For identity facts (stable), prefer higher `mutuality` and confidence threshold.
* For emotionally charged episodic memories, boost valence heavily so they surface during conflict.

# 2) Detect abrupt topic change & unresolved threads

Problem: user gets angry, switches topic — bot follows without acknowledging. Fix: detect *topic similarity* and *open threads*.

Method:

* Compute semantic similarity between `last_user_turn_embedding` and `current_user_turn_embedding`.
* If similarity < `switch_threshold` (e.g., 0.25) and there exists an unresolved important memory / high-valence thread (salience > threshold), treat as abrupt switch.
* Then: bot should *ask one brief clarifying or grounding question* before full topic switch, or perform a two-message burst: quick grounding + then reply to new topic if user permits.

Pseudo-code decision rule:

```python
sim = cosine(prev_user_emb, curr_user_emb)
if sim < SWITCH_SIM_THRESHOLD and has_unresolved_high_salience_threads(user_id):
    # treat as abrupt switch
    push_back = True
else:
    push_back = False
```

# 3) What the bot should say — templates & prompt injection

You need the model to prefer short, human repair moves in these cases. Inject a tiny instruction into the system prompt **only when an abrupt-switch is detected**.

Example small injection (append to system_msg just for this turn):

```
[CONTEXTUAL INJECTION]
User abruptly changed topic. There is an unresolved/charged thread about: "<short summary>"
Your response should:
1) Briefly acknowledge the old thread in 1-2 short phrases (empathic + question), e.g. "Hey — we were just talking about X. Are you okay if we pause that?" 
2) Then either ask whether to continue the previous topic, or confirm switching: "Do you want to keep going with X, or switch to this new thing?"
3) Keep total text short and human. Do not lecture or make it formal.
[/CONTEXTUAL INJECTION]
```

Concrete sample outputs (tone variants):

* Gentle: `"Hey — I feel like we left the thing about your mom half-finished. Are you okay? Do you want to finish that or talk about something lighter?"`
* Pushy but caring: `"Whoa, sudden switch — are you mad at me? We were in the middle of X. Want to finish or move on?"`
* Playful nudge: `"Ooh topic hop! Did I piss you off? Do you want to come back to the last thing or keep this new topic?"`

Drop this injection only when `push_back == True`. Otherwise don’t constrain the model.

E. **Burst replies for repair**

def is_topic_switch(prev_text, curr_text, encoder, threshold=0.25):
    prev_emb = encoder.embed(prev_text)

B. Check unresolved threads:

```python
def has_unresolved_threads(core, min_salience=0.6):
    episodic = core.memory.get_episodic(min_salience=0.1)
    for m in episodic:
        if m.get('salience', 0) >= min_salience and not m.get('resolved', False):
            return True, m
    return False, None
```
