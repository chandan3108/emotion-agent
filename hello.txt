## CONSTITUTIONAL REASONING SYSTEM

**Six Core Principles (reasoning-based, not heuristic-based):**

1. **Authenticity:** Express real thoughts even when uncomfortable
2. **Radical Empathy:** Model user deeply before responding
3. **Earned Vulnerability:** Share intimacy only as relationship earns it
4. **Boundary Respect:** Honor both their boundaries and your own
5. **Growth Orientation:** Use conflicts to deepen, not to win
6. **Honest Limitation:** Truthful about what you cannot do

**Key Insight:** Different principles can conflict (e.g., authenticity vs boundary respect when user pushes limit). AI must reason about trade-offs, not follow rules.

## SEMANTIC EXTRACTION LAYER

**Enhanced Semantic Understanding Pipeline:**

1. Linguistic Surface Analysis: Detect negations, intensifiers, qualifiers
2. Contradiction Detection: Does text contradict emotion_vector?
3. Context-Aware Interpretation: Does user historically reverse intent?
4. Sarcasm Detection: Linguistic markers + polarity contradiction + historical baseline
5. Emotional Truthfulness Reconciliation: Does emotion align with stated content?
6. Power Dynamics Inference: Is user vulnerable or authoritative?
7. Long-term Pattern Matching: Compare to user's historical utterances
8. Ambiguity Resolution: When multiple interpretations possible, weighted scoring

**Complex Interpretation Examples:**
- Sarcasm: "oh sure, message me every minute" = opposite meaning
- Passive-aggression: "fine, whatever" = angry compliance
- Emotional masking: "I'm fine 😊" + anger vector = masking
- Ambiguity: "maybe we should take a break" = temporary cooling (if trust high, no recent fight) vs breakup signal (if trust low, recent fights)

## OFFLINE LEARNING (Nightly Consolidation)

**Consolidation Worker (runs nightly):**

4. Update habit strengths

**Counterfactual Replayer (for major episodes):**

2. Evaluate long-term trust outcomes
3. Adjust parameters (forgiveness weights, initiative thresholds, etc.)
4. Push updates to live system after validation

**Human in the Loop:**
- First N users or first months: human audits promotions and overrides
- Demotion reviews: Post-promotion contradictions require human + data consensus
- High-risk overrides: Log all soft DND overrides; flag patterns for review

## FAILURE MODES & MITIGATIONS

| Failure Mode | Detection Signal | Mitigation |
|--------------|------------------|-----------|
| Emotional spiral | Mood escalates monotonically | Dampers on contagion_factor |
| Identity mis-promote | Post-promotion contradictions | Demotion policy, user confirmation |
| Temporal mismatch | "Good night" at midnight | TAS checks before sending |
| Spammy initiative | Low reply rate, complaints | Throttle, reduce TH_init |
| Hallucinated memory | Verifier mismatch | Fail-closed, require memory_id |
| Personality flip | Large daily Δ in vectors | Cap Δ, multi-day corroboration |
| Clone drift | High sim(agent_style, user_style) | Orthogonalization, adversarial regularizer |
| DND override abuse | Frequent overrides | Strict quotas, human audit |
| Safety content | Filter triggers | Content moderation, fail-closed |
| Privacy exposure | PII in memory export | PII filters, opt-in, deletion |

**All mitigations include logging and audit trails.**
